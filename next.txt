What Works
Feature
Functions — params, return types, #[vertex]/#[fragment]/#[compute(N,N,N)]
Local vars — let, let mut, type annotation, init
Const globals — const NAME: T = expr; across functions
If/else, loop — control flow
Binary/unary ops — arithmetic, comparison, logical, bitwise
Casts — expr as Type
Structs — definition, Name { field: expr, ... } literals, .field access (read+write), nested, return type

Enums (payload) — enum Option { Some(f32), None }, Option::Some(1.0), stored as OpTypeStruct{u32,payload}
Descriptors — #layout(set=N,binding=M) uniforms, #layout(_push_constant)
Builtins — _vertex_id
SPIR-V output — valid header, instructions assembled
Known Bugs
#	Bug
1	break/continue don't advance token — the match arms at parser.rs:679-680 return the variant without calling self.advance()
2	Comparison ops return LHS type not bool — infer_expr_type for Binary always clones lhs.ty instead of emitting Type::Scalar(Bool) for comparisons
3	else if whitespace-sensitive — lexer token #[token("else if")] requires exactly one space; else\nif or else  if lexes as separate Else + If with no ElseIf token
4	Array element type broken — Index expression type inference returns base type not element type; ArrayDeclaration hits NotSupported in resolver; from_name maps unknown names to Struct not Array
5	Descriptors always Uniform — StorageClass::Descriptor hardcoded to spirv::StorageClass::Uniform; no StorageBuffer path
6	SPIR-V version hardcoded to 1.0 — set_version(1, 0) restricts feature set
7	No SPIR-V validation in tests — check_spv only checks magic number and length
Missing Features
Feature	Notes
match expression	No way to destructure enum payloads — you can create Option::Some(1.0) but can't extract the 1.0
while loop	Only loop { } with break/continue
Arrays — [T; N] type syntax, indexing, array literals	Partial parsing exists but resolver/typechecker/codegen reject or mis-handle it
else if fix	Needs lexer change to handle arbitrary whitespace (e.g. regex else[ \t\n]+if with callback that normalizes)
Enum match	Required for payload-bearing enums to be useful beyond pass-through
Pointers/references	No &, *, or explicit pointer types for users
Generics / polymorphism	Everything monomorphized manually
Module system	Single-file only; no imports
Error recovery / diagnostics	Parse errors stop at first issue, no suggestions
Multiple return types / tuples	Only single return type
SPIR-V validation pass	Could call spirv-val or use rspirv::validation (if available) in tests
